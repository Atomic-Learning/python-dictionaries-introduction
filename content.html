Creating Dictionaries

in Python, Dictionaries use curly braces {}. A key and its corresponding value are separated by a colon :, and key-value pairs are separated by commas ,:

atomic_numbers = {"H": 1, "He": 2, "Li": 3} student_grades = {"Alice": 95, "Bob": 87, "Charlie": 92} empty_dict = {} print(atomic_numbers) print(student_grades) print(empty_dict)

When defining a dictioanry with multiple key-value pairs, you can spread the pairs across multiple lines for readability. Indenting each pair is a common style convention (though not required by Python):

person = { "name": "Alice", "age": 30, "city": "New York" } print(person)

Accessing Values by Key

Use square brackets with the key to get the corresponding value:

atomic_numbers = {"H": 1, "He": 2, "Li": 3} print(atomic_numbers["H"]) print(atomic_numbers["Li"]) grades = {"Alice": 95, "Bob": 87} print(grades["Alice"])

Keys and Values Can Be Different Types

Values can be any type. Keys are more restrictive in terms of what type can be used - strings and numeric values are most common (the technical requirement is that keys must be hashable). Within these constraints, a single dictionary can have a mixture of key and value types:

mixed_dict = { 10: 4.2, # integer key, float value "string key" : 8 # string key, integer value } print(mixed_dict[10]) print(mixed_dict["string key"])

Accessing Non-Existent Keys

If you try to access a key that doesn't exist, Python raises a KeyError:

grades = {"Alice": 95, "Bob": 87} print(grades["David"])
